# ADR-008 — JWT: HS256 no MVP, Migração para RS256 na Extração

**Data:** 2026-07-14  
**Status:** Aceito (com revisão programada)  
**Decisores:** Lucas (Engenheiro), Claude (Tech Lead / Arquiteto)

---

## Contexto

O sistema precisa assinar e validar JWTs. Existem dois algoritmos principais:

- **HS256 (HMAC-SHA256):** Algoritmo simétrico. Uma única chave secreta é usada para assinar e verificar.
- **RS256 (RSA-SHA256):** Algoritmo assimétrico. Chave privada assina; chave pública verifica.

## Decisão

**MVP:** HS256 com chave de no mínimo 256 bits (32 bytes), validada em startup via `@PostConstruct`.

**Produção / Extração de microsserviços:** migrar para RS256.

## Análise

### HS256
- Prós: Simples, sem gerenciamento de par de chaves, adequado para sistema monolítico onde apenas um processo assina e valida
- Contras: Se o segredo vazar, todos os tokens históricos são comprometidos. Em microsserviços, o segredo deve ser compartilhado com todos os serviços que validam — cada serviço com o segredo é um vetor de ataque.

### RS256
- Prós: Chave privada fica apenas no IAM; outros serviços usam a chave pública (`/.well-known/jwks.json`). Comprometimento de um serviço não compromete a capacidade de assinar tokens.
- Contras: Gerenciamento de par de chaves; rotação mais complexa.

## Consequências

**Agora (HS256):**
- `JWT_SECRET` deve ter no mínimo 32 caracteres (256 bits) — validado em startup
- A chave não deve aparecer em logs, git history ou variáveis de ambiente legíveis
- Rotação de chave = todos os tokens existentes tornam-se inválidos (logout forçado de todos os usuários)

**Quando extrair IAM como microsserviço:**
1. Gerar par RSA (4096 bits recomendado)
2. Chave privada: variável de ambiente ou AWS Secrets Manager, somente no IAM
3. Chave pública: exposta em `GET /api/v1/auth/.well-known/jwks.json`
4. Outros serviços: configurados apenas com a URL do JWKS endpoint
5. Implementar rotação de chaves com período de overlap (N e N-1 válidas simultaneamente)

## Riscos Conhecidos

- Sem rotação automatizada de chave no MVP — se comprometida, exige intervenção manual
- Mitigação: TTL curto do access token (15min) limita a janela de exposição
